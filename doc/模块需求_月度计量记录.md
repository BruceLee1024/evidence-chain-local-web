# 模块需求：月度进度计量确认单

> 版本：v1.0
> 优先级：🟡 P1
> 对应需求文档：需求文档_v1.0.md — 4.4 节

---

## 一、定位

月度进度计量确认单是**甲方签字确认的产值文件**。在结算时，甲方只认签字确认的产值，这是施工单位申报进度款和结算的**核心依据**。本模块负责记录每个月的计量确认情况，留存签字确认单原件。

---

## 二、数据模型

```typescript
月度计量记录 {
  // === 基本信息 ===
  id:               string        // 系统自动生成
  项目:              string        // 所属项目
  
  // === 计量信息 ===
  计量月份:            string        // "2026-06"，必填，格式 YYYY-MM
  本期完成产值:         number        // 万元或元，必填
  累计完成产值:         number        // 万元或元，必填
  
  // === 支撑文件 ===
  确认单文件:           File[]        // 甲方签了字的确认单扫描件/照片（核心文件）
  计量明细表:           File[]        // Excel 或 PDF 明细（辅助文件）
  
  // === 签认信息 ===
  甲方签字人:           string
  确认日期:            string        // YYYY-MM-DD，甲方实际签字日期
  备注:               string
}
```

### 关键设计决策

1. **计量月份**：格式严格为 `YYYY-MM`，不允许跨月记录。例如 2026 年 6 月的计量，月份填 `2026-06`
2. **产值单位**：不做单位转换，用户在表单中自行约定统一单位。建议在表单下方加提示：「请与合同约定的产值单位保持一致」
3. **确认单文件**：最核心的文件，**建议必传**（如果没有确认单，这记录的意义就打折了）
4. **不设人工校验**：本期产值和累计产值的逻辑关系（本期 = 累计 - 上期累计）由用户自己保证，系统不做自动计算

## 三、功能需求

### 3.1 录入表单

出现在证据管理页中，与现有 Tab 并列。

**表单字段布局**：

```
┌─────────────────────────────────────────────┐
│  月度计量确认单归档                            │
├─────────────────────────────────────────────┤
│  计量月份 *     [  2026-06        ]         │
│                 (格式：YYYY-MM)              │
│  确认日期 *     [  2026-07-05     ]         │
│  本期完成产值 *  [  380000         ]  元    │
│  累计完成产值 *  [  1650000        ]  元    │
│  甲方签字人     [  王经理                 ]  │
├─────────────────────────────────────────────┤
│  备注                                        │
│  [ 3#楼已施工至主体10层                     ] │
├─────────────────────────────────────────────┤
│  📎 签字确认单  [选择文件]  (必传，支持多文件)  │
│  📎 计量明细表  [选择文件]  (辅助文件)         │
├─────────────────────────────────────────────┤
│  [ 💾 保存月度计量记录 ]                     │
└─────────────────────────────────────────────┘
```

**校验规则**：
- 计量月份：必填，格式 `YYYY-MM`
- 确认日期：必填
- 本期完成产值：必填，> 0
- 累计完成产值：必填，> 0
- 签字确认单：建议必传（soft required — 没有确认单可以保存，但会显示提示 "建议上传签字确认单"）

### 3.2 列表展示

右侧证据列表新增月度计量记录条目的展示：

```
🟡 月度计量 · 2026-06 · 产值38万 · 2个附件
```

**排序**：按计量月份降序（最近的月份在前）。

**特殊显示**：列表中的月度计量记录会显示一个「趋势指示」：
- ✅ 比上期增长
- ✅ 与上期持平
- ⚠️ 比上期下降

> v1.0 简化实现：不真正计算趋势，仅作为 UI 上的预留占位。

### 3.3 交互流程

```
用户在证据管理页
  → 点击「月度计量记录」Tab
  → 看到表单（新建模式）
  → 填写信息 + 上传文件
  → 点击保存
  → 表单清空，右侧列表刷新
  → 新记录出现在列表中
```

> 与材料进场记录一致，v1.0 暂不支持编辑和删除。

## 四、后端 API

### 4.1 新建月度计量记录

```
POST /api/projects/:projectId/monthly-measurements

Content-Type: multipart/form-data

字段：
  - month (string, required) — 计量月份，格式 YYYY-MM
  - confirmDate (string, required) — 确认日期
  - currentValue (number, required) — 本期完成产值
  - cumulativeValue (number, required) — 累计完成产值
  - ownerSigner (string) — 甲方签字人
  - note (string) — 备注
  - confirmFiles (file[], multiple) — 签字确认单文件
  - detailFiles (file[], multiple) — 计量明细表文件

响应：201 Created
  {
    id: string,
    type: 'monthly',
    title: string,        // 自动生成："2026-06 月度计量"
    location: '',         // 不关联部位
    evidenceDate: string, // = confirmDate
    amount: number,       // = currentValue
    payload: { month, confirmDate, currentValue, cumulativeValue, ownerSigner, note },
    files: [...]
  }
```

### 4.2 查询月度计量记录

```
GET /api/projects/:projectId/evidence?type=monthly
```

## 五、与现有系统的集成

### 5.1 证据管理页 Tab

与材料进场记录一同新增，作为第四个 Tab。

### 5.2 项目主页概览

Dashboard 增加 Metric 卡片：

```
Metric(label="月度计量记录", value={counts.monthly || 0}, tone="orange")
```

Dashboard 的「最近证据」列表中也会自然展示最新月度计量记录。

### 5.3 结算组卷匹配

月度计量记录的匹配规则：

- **时间匹配**：计量月份（`evidenceDate = confirmDate`）在结算项的时间范围内，或计量月份对应的整月在结算项时间范围内
- **部位匹配**：月度计量记录无部位字段。改为用备注中的关键词或项目级匹配
- **金额关联**：结算项的金额与月度计量的 `currentValue` 或 `cumulativeValue` 接近时，自动推荐（差异 < 10% 时置信度 85，否则 50）
- 匹配置信度：68（时间吻合），85（时间吻合 + 金额接近）

匹配结果展示：

```
结算项：3#楼地下室底板混凝土
  ✓ 月度计量 2026-06 · 产值38万元 · 2026-07-05确认  [查看] [移除]
  ⚠️ 候选：月度计量 2026-05 · 产值25万元 · 差异较大  [查看] [确认]
```

### 5.4 导出集成

导出结算证据包时：

1. 月度计量确认单文件放入对应结算项的子文件夹
2. 结算项清单.xlsx 中增加一列「本期产值」，从关联的月度计量记录中提取

## 六、数据库

### 6.1 表结构

复用现有的 `evidence` 表和文件表，无需新增表。

**evidence 表新增字段说明**：
- `type = 'monthly'` 标识月度计量记录
- `title` 自动生成为 `"{计量月份} 月度计量"`
- `location = ''` (空字符串)
- `evidence_date = confirmDate` (确认日期)
- `amount = currentValue` (本期完成产值，同时作为 evidence 表的 amount 字段，便于结算组卷中按金额匹配)
- `end_date = ''` (空字符串，不涉及结束日期)
- `payload_json` 存储以下字段：month, confirmDate, currentValue, cumulativeValue, ownerSigner, note

### 6.2 文件角色

| 上传字段 | 文件角色 |
|---------|---------|
| confirmFiles | `confirmation` |
| detailFiles | `detail` |

## 八、与材料进场记录的区别对照

| 维度 | 材料进场记录 | 月度计量记录 |
|------|------------|------------|
| 核心目的 | 证明品牌/规格/数量合规 | 证明甲方认可产值 |
| 关键文件 | 合格证、验收照片 | 签字确认单 |
| 匹配依据 | 时间 + 材料名称模糊匹配 | 时间 + 金额接近度 |
| 涉及金额 | 无 | 有（本期产值） |
| 部位关联 | 无 | 无 |
| 导出作用 | 证明材料合规 | 证明产值得到确认 |
