## 第二轮：证据类型范围

### Hermes 提出的 5 类核心证据（按优先级排序）

| 优先级 | 证据类型 | 理由 |
|--------|---------|------|
| 🔴 P0 | 签证变更单 | 丢了亏最大 |
| 🔴 P0 | 隐蔽工程验收记录 + 照片 | 无法重取证 |
| 🟡 P1 | 材料进场验收记录 | 品牌/规格争议高发 |
| 🟡 P1 | 月度进度计量确认单 | 甲方签字才认账 |
| 🟢 P2 | 联系函 / 会议纪要 | 辅助证据 |

### 决策
- 不搞大而全的文档管理
- 只管这 5 类，边界清晰
- ✅ 用户同意

### 隐蔽工程照片方案
- 手机拍 → 自动加时间戳水印
- 拍完填 3 个字段：部位/工序/说明（≤10 秒）
- 自动归类
- ✅ 用户同意

### 结算证据包方案
- 导入结算项 Excel → 自动匹配 → 人工校对 → 一键导出
- ✅ 用户同意

### 系统形态
- **第一阶段**：Web 端
- **第二阶段**：移动端拍照
- ✅ 用户同意

## 第四轮：数据模型定义

### 4 类证据的数据结构

| 证据类型 | 核心字段 |
|---------|---------|
| 签证变更单 | 编号、类型、原因、增减金额、三方签字人、日期、附件 |
| 隐蔽工程记录 | 部位、工序、验收日期、照片列表（时间戳水印）、验收结论 |
| 材料进场记录 | 材料名称、规格型号、数量、品牌、供应商、合格证文件 |
| 月度计量记录 | 计量月份、本期产值、累计产值、确认单文件、甲方签字人 |

### 关联关系
各证据通过「项目 + 部位 + 时间」关联，共同支撑结算项

- ✅ 用户确认通过

---

## 第五轮：用户操作流程

### 统一录入流程
```
选择项目 → 选择证据类型 → 填基本信息 → 上传文件 → 保存
```

### 关键决策
- **签证单只做录入归档，不做线上审批流**（签字走线下）→ 用户同意
- **部位选择方式**：方案 C（既支持项目结构树选择，也支持手动输入模糊匹配）→ 用户同意

### 结算组卷 4 步流程
```
① 导入结算项（Excel）→ ② 智能匹配 → ③ 人工校对 → ④ 一键导出
```

## 第六轮：导入结算项 — 8 大场景决策

| # | 场景 | Hermes 判断 | 最终决策 |
