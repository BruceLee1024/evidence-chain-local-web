# 模块需求：材料进场记录

> 版本：v1.0
> 优先级：🟡 P1
> 对应需求文档：需求文档_v1.0.md — 4.3 节

---

## 一、定位

材料进场记录是结算证据链的重要组成部分。施工过程中，材料进场时的**品牌、规格、数量、合格证明**是结算时与甲方/分包对账的常见争议点。本模块负责记录每次材料进场的关键信息，并留存证明文件。

---

## 二、数据模型

```typescript
材料进场记录 {
  // === 基本信息 ===
  id:               string        // 系统自动生成
  项目:              string        // 所属项目（从当前项目自动带入）
  
  // === 材料信息 ===
  进场日期:           string        // YYYY-MM-DD，必填
  材料名称:            string        // 必填，如 "HRB400 钢筋"
  规格型号:            string        // 如 "Φ25"
  单位:               string        // 必填，如 吨 / m³ / ㎡ / 个 / 米
  进场数量:            number        // 必填，正数
  品牌厂家:            string        // 品牌或生产厂家
  供应商:             string        // 供应商名称
  
  // === 文件证据 ===
  合格证文件:           File[]        // 合格证、检测报告等证明文件
  验收照片:            File[]        // 进场验收时的现场照片
  
  // === 签收信息 ===
  收货人:             string        // 现场收货人/材料员
  备注:               string
}
```

### 关键设计决策

1. **单位字段**：预置常用选项（吨、m³、㎡、个、米、千克、箱、根），同时也支持手动输入其他单位
2. **进场数量**：只做正数校验，不做库存计算（本系统不是仓库管理系统）
3. **合格证文件 vs 验收照片**：分开上传，便于后续导出时分类归档
4. **不关联部位**：材料进场往往对应整个标段/楼栋，不像隐蔽工程那样精确到具体部位。如有需要可在备注中说明

## 三、功能需求

### 3.1 录入表单

出现在证据管理页中，与现有「签证变更单」「隐蔽工程记录」并列为一个 Tab。

**表单字段布局**：

```
┌─────────────────────────────────────────────┐
│  材料进场记录归档                              │
├─────────────────────────────────────────────┤
│  进场日期 *     [  2026-06-15  ]             │
│  材料名称 *     [  HRB400 钢筋              ] │
│  规格型号       [  Φ25                     ] │
│  单位 *         [  吨 ───────────── ▼      ] │
│  进场数量 *     [  120                     ] │
│  品牌厂家       [  首钢                    ] │
│  供应商         [  XX钢铁有限公司            ] │
│  收货人         [  张三                    ] │
├─────────────────────────────────────────────┤
│  备注                                        │
│  [ 3#楼地下室底板用                        ] │
├─────────────────────────────────────────────┤
│  📎 合格证/检测报告  [选择文件]  (支持多文件)   │
│  📎 验收照片         [选择文件]  (支持多文件)   │
├─────────────────────────────────────────────┤
│  [ 💾 保存材料进场记录 ]                     │
└─────────────────────────────────────────────┘
```

**校验规则**：
- 进场日期：必填
- 材料名称：必填，去除首尾空格
- 单位：必填，从预置列表选或手动输入
- 进场数量：必填，> 0
- 合格证/验收照片：至少上传一种（可以只传照片不要合格证，也可以只有合格证没照片）

### 3.2 列表展示

证据管理页右侧的「P0 证据列表」扩展为「P0/P1 证据列表」，增加材料进场记录的展示。

**每一条记录展示**：
```
🟡 材料进场 · HRB400 钢筋 Φ25 · 120吨 · 2026-06-15 · 2个附件
```

**排序**：按进场日期降序，同日期按创建时间降序。

### 3.3 交互流程

```
用户在证据管理页
  → 点击「材料进场记录」Tab
  → 看到表单（新建模式）
  → 填写信息 + 上传文件
  → 点击保存
  → 表单清空，右侧列表刷新
  → 新记录出现在列表中

用户点击列表中的某条记录
  → 右侧/弹出详情，显示完整信息
  → 可查看/下载附件的链接
```

> v1.0 暂不支持「编辑已有记录」和「删除记录」功能（与现有签证变更单、隐蔽工程记录保持一致，仅支持新增和查看）。

## 四、后端 API

### 4.1 新建材料进场记录

```
POST /api/projects/:projectId/material-records

Content-Type: multipart/form-data

字段：
  - entryDate (string, required) — 进场日期
  - materialName (string, required) — 材料名称
  - spec (string) — 规格型号
  - unit (string, required) — 单位
  - quantity (number, required) — 进场数量
  - brand (string) — 品牌厂家
  - supplier (string) — 供应商
  - receiver (string) — 收货人
  - note (string) — 备注
  - certificates (file[], multiple) — 合格证/检测报告
  - photos (file[], multiple) — 验收照片

响应：201 Created
  {
    id: string,
    type: 'material',
    title: string,        // 自动生成："HRB400 钢筋 Φ25"
    location: '',         // 材料进场不关联部位，空字符串
    evidenceDate: string,
    amount: null,         // 材料进场不涉及金额
    payload: { materialName, spec, unit, quantity, brand, supplier, receiver, note },
    files: [...]
  }
```

### 4.2 查询材料进场记录

材料进场记录通过通用证据查询接口获取：

```
GET /api/projects/:projectId/evidence?type=material
```

与现有 evidence 查询接口共用，type=material 即可筛选。

## 五、与现有系统的集成

### 5.1 证据管理页 Tab

现有代码中，证据管理页有两个 Tab：

```
evidenceLabels = {
  variation: '签证变更单',
  hidden: '隐蔽工程记录',
  supplemental: '后补资料'
}
```

改为四个：

```typescript
evidenceLabels = {
  variation: '签证变更单',
  hidden: '隐蔽工程记录',
  material: '材料进场记录',
  monthly: '月度计量记录',
  supplemental: '后补资料'
}
```

在 EvidencePage 组件中增加两个 Tab 按钮。

### 5.2 项目主页概览

Dashboard 的 Metric 卡片增加两个：

```
Metric(label="材料进场记录", value={counts.material || 0}, tone="purple")
Metric(label="月度计量记录", value={counts.monthly || 0}, tone="orange")
```

### 5.3 结算组卷匹配

材料进场记录**参与结算组卷的匹配逻辑**：

- 在 `matching.js` 的 `matchEvidenceForItems` 中，材料进场记录按以下规则匹配：
  - **时间匹配**：进场日期在结算项的时间范围内
  - **部位匹配**：材料进场记录无部位字段，跳过部位匹配。改为用材料名称关键词模糊匹配结算项名称
  - 匹配置信度：68（时间吻合 + 材料名称相似）

### 5.4 搜索功能

材料进场记录已自动被全局搜索覆盖（搜索在 `searchEvidence` 中按 title/location/payload_json 全文匹配）。

## 六、数据库

### 6.1 表结构

复用现有的 `evidence` 表和 `files`/`evidence_files` 表，无需新增表。

**evidence 表新增字段说明**：
- `type = 'material'` 标识材料进场记录
- `title` 自动生成为 `"{材料名称} {规格型号}"`
- `location = ''` (空字符串，不关联部位)
- `evidence_date = entryDate` (进场日期)
- `amount = null` (不涉及金额)
- `payload_json` 存储以下字段：materialName, spec, unit, quantity, brand, supplier, receiver, note

### 6.2 文件角色

| 上传字段 | 文件角色 |
|---------|---------|
| certificates | `certificate` |
| photos | `photo` |
