> 版本：v1.0

## 一、项目背景与目标

### 1.1 项目背景

施工单位在工程结算过程中，经常面临以下问题：

- **结算时证据找不到**：隐蔽工程照片、签证变更单、材料进场记录等关键证据散落在微信群、电脑文件夹、纸质档案中，结算时翻箱倒柜也凑不齐
- **资料整理混乱**：没有统一的分类体系和归档标准，同一项目的证据分散在不同人手里
- **与甲方/分包对账扯皮**：拿不出带时间戳的现场照片、没有完整的结算证据包，导致该要的钱要不回来

### 1.2 项目目标

构建一套**面向施工单位商务结算人员的证据链管理工具**，旨在：

1. **全过程留证**：从开工到结算，关键节点不遗漏
2. **快速定位资料**：搜索即达，不用翻文件夹
3. **一键组卷导出**：结算时自动匹配证据，导出完整结算证据包

### 1.3 产品定位

> **不是「文档管理系统」**，而是**「证据链管理工具」**——聚焦结算时最有争议的几类证据，帮商务结算人员提效。

证据链管理系统 v1.0

│   ├── 隐蔽工程记录（录入 + 照片批量上传 + 列表）
│   ├── 材料进场记录（录入 + 列表）
│   └── 月度计量记录（录入 + 列表）

├── 📦 结算组卷

### 3.2 证据类型范围

| 优先级 | 证据类型 | 说明 |
|--------|---------|------|
| 🔴 P0 | **签证变更单** | 结算时争议最大、丢一个亏最大 |
| 🔴 P0 | **隐蔽工程验收记录 + 照片** | 无法重新取证，时间戳关键 |
| 🟡 P1 | **材料进场验收记录** | 品牌/规格/数量不符的争议高发区 |
| 🟡 P1 | **月度进度计量确认单** | 甲方签字才认账 |
| 🟢 P2 | 联系函 / 会议纪要 | 辅助证据，先存着 |

## 四、数据模型定义

### 4.1 签证变更单

```typescript
签证变更单 {
  id:               string        // 系统自动生成
  项目:              string        // 所属项目
  变更编号:            string        // 纸质单上的编号
  变更类型:            '设计变更' | '现场签证' | '工程联系单'
  变更原因:            string        // 简短描述
  增减金额:            number        // 单位：元，正数增负数减
  是否涉及工期:          boolean
  施工单位签字人:        string
  监理签字人:           string
  甲方签字人:           string
  签证日期:            string        // YYYY-MM-DD
  附件列表:            File[]        // 扫描件、照片、图纸
  备注:               string
}
```

### 4.2 隐蔽工程记录

```typescript
隐蔽工程记录 {
  id:               string
  项目:              string
  部位:              string        // e.g. "3#楼-2层-柱KL1"
  工序:              string        // e.g. "钢筋绑扎" / "模板安装" / "混凝土浇筑"
  验收日期:           string
  
  照片列表: [
    {
      照片文件:        File
      拍摄时间:        string        // 自动加时间戳水印
      拍摄人:          string
      备注:           string        // 可选：简要说明
    }
  ]
  
  验收单文件:          File          // 签字验收单
  验收结论:            '合格' | '不合格' | '需整改'
  备注:               string
}
```

### 4.3 材料进场验收记录

```typescript
材料进场记录 {
  id:               string
  项目:              string
  进场日期:           string
  材料名称:            string        // e.g. "HRB400 钢筋"
  规格型号:            string        // e.g. "Φ25"
  单位:               string        // 吨 / m³ / ㎡ / 个
  进场数量:            number
  品牌/厂家:           string
  供应商:             string
  合格证/检测报告:       File[]
  验收照片:            File[]
  收货人:             string
  备注:               string
}
```

### 4.4 月度进度计量确认单

```typescript
月度计量记录 {
  id:               string
  项目:              string
  计量月份:            string        // "2026-06"
  本期完成产值:         number        // 元
  累计完成产值:         number
  确认单文件:           File          // 甲方签字扫描件
  计量明细表:           File          // Excel 或 PDF 明细
  甲方签字人:           string
  确认日期:            string
  备注:               string
}
```

### 4.5 数据之间的关联关系

各证据类型通过「项目 + 部位 + 时间」三个维度关联，共同支撑结算项：

```
结算项：「3#楼地下室底板混凝土」
  ├── 签证变更单  #2026-023  (因基坑超挖增加混凝土量)
  ├── 隐蔽工程记录 2026-05-15 (底板钢筋验收 + 照片3张)
  ├── 材料进场记录 2026-05-10 (C35混凝土120m³)
  └── 月度计量 2026-06 (本期已确认该部位产值)
```

## 五、核心流程

### 5.1 系统操作总流程

```
打开系统
  │
  ├── ① 选择当前项目
  │
  ├── ② 日常工作（施工阶段）
  │   ├── 上传新签证单
  │   ├── 新增隐蔽工程记录 + 照片
  │   ├── 登记材料进场
  │   └── 归档月度计量单
  │
  └── ③ 结算阶段
      ├── 导入结算项（从 Excel 粘贴/上传）
      ├── 系统自动匹配证据（按部位+时间）
      ├── 人工校对补漏
      └── 一键导出结算证据包
```

### 5.2 日常录入流程（统一交互模式）

```
选择项目 → 选择证据类型 → 填基本信息 → 上传文件 → 保存
```

- **签证变更单**：走线下签字 → 系统只做录入归档，不做线上审批流
- **隐蔽工程记录**：支持批量上传多张照片，自动打时间戳水印。部位选择支持「从项目结构树选」和「手动输入模糊匹配」两种方式
- **材料进场**：常规表单录入 + 文件上传
- **月度计量**：常规表单录入 + 文件上传

### 5.3 结算组卷流程

```
① 导入结算项
  从 Excel 复制/粘贴或上传文件
  → 自动解析，支持手动选列匹配（解决表头不统一问题）
  → 系统记住匹配规则，下次自动识别

② 智能匹配证据
  分三层匹配：
  - 第一层：精确匹配（部位完全一致）
  - 第二层：模糊匹配（部位关键词包含）
  - 第三层：时间关联（施工时段匹配）
  匹配结果分三档显示：
  ✓ 已匹配 → 绿色
  ⚠️ 部分匹配 → 黄色（需人工确认）
  ✗ 无匹配 → 红色（需手动补传）

③ 人工校对
  - 匹配对的 → 保留
  - 匹配错的 → 移除
  - 缺证据的 → 手动补传（标注「后补资料」）
  - 同一文件支持被引用到多个结算项（只存一份原件）

④ 一键导出
  默认格式：按结算项归档的文件夹结构
  每个结算项一个子文件夹，内含所有支撑文件
  + 附带结算项清单 Excel
```

## 六、导入结算项 — 场景决策

| 场景 | 优先级 | 方案 |
|------|--------|------|
| **① Excel 格式不统一** | ✅ v1.0 必做 | 上传后预览前 3 行，用户手动选列匹配，系统记住规则 |
| **② 部位信息不全** | ✅ v1.0 必做 | 精确搜 → 搜不到显示无匹配 → 手动关联 + 记录相似度 |
| **③ 一个证据支持多个结算项** | 🔶 v1.0 简化做 | 自动显示匹配情况，但不做手动跨项推荐 |
| **④ 结算项找不到匹配** | ✅ v1.0 必做 | 显示 ⚠️ 无匹配 → 可补传 + 标注「后补资料」 |
| **⑤ 混了多个项目** | ⏰ v1.1 | v1.0 加提示「请确认只包含一个项目」 |
| **⑥ 同一文件匹配到多个结算项** | ✅ v1.0 必做 | 只存一份原件，多处引用 |
| **⑦ 甲方要特定格式** | 🔶 v1.0 简化做 | 默认只做「按结算项归档」一种格式 |
| **⑧ 历史数据导入** | ⏰ v1.1 | 单独做迁移工具，与核心功能解耦 |

## 七、全局搜索

**定位**：解决"我记得有这个签证单，但翻不到了"的问题

**搜索维度**：
- 关键词搜索（签证编号、材料名称、部位名称等）
- 按项目筛选
- 按证据类型筛选（签证/隐蔽/材料/计量）
- 按日期范围筛选

**搜索结果**：卡片式展示，显示摘要信息 + 所属项目 + 日期
